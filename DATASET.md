# Dataset Description

## Source

This project uses a **synthetic dataset** generated by `create_sample_data.py`. Synthetic data was chosen because:
- Public Enron dataset attachments are hard to extract cleanly
- Synthetic data gives us controlled test cases for evaluation
- All examples are realistic business correspondence (vendor approvals, budget reviews, contract renewals)

The format follows real `.eml` standards (RFC 5322 compliant) with proper headers (`Message-ID`, `In-Reply-To`, `References`, `Date`, `From`, `To`, etc.).

## Selection / Generation

- **3 threads** simulating realistic enterprise scenarios:
  1. Storage Vendor Approval (4 messages, 2 PDFs)
  2. Q2 Budget Review (2 messages, 1 PDF)
  3. Vendor Contract Renewal (2 messages, 1 PDF)
- **Date range:** April – June 2001 (matches Enron-era timeframe)
- **Senders/Recipients:** Generic enterprise roles (CFO, CTO, procurement, etc.)

## Final Counts

| Metric | Count |
|---|---|
| Threads | 3 |
| Messages | 8 |
| PDF Attachments | 4 |
| Total indexed chunks | ~25-35 |
| Approximate text size | < 1 MB |

## Preprocessing

- Email bodies: signature/quoted-text stripping not applied (synthetic data is already clean)
- PDFs: page-level extraction using PyMuPDF; 300-token chunks with 50-token overlap
- Whitespace normalization

## Switching to Real Enron Data

To use the real Enron dataset:
1. Download from https://www.kaggle.com/datasets/wcukierski/enron-email-dataset
2. Pick one mailbox folder (e.g., `maildir/lay-k/`)
3. Convert messages to `.eml` format and place in `data/raw/`
4. Re-run `python ingest.py`

The Enron emails are mostly text-only (no attachments). To test the attachment pipeline, use the synthetic dataset OR add your own PDFs.
